Frequently Asked Questions

Q: How did you break down the project?
A: The project was broken down into several key phases:
  • Choosing the API and testing endpoints with Postman.
  • Designing wireframes and mockups using Figma.
  • Implementing and developing the website using Golang, HTML, and CSS.
  • Testing and debugging the application.
  • Final delivery of the solution.
Q: How did you distribute tasks among team members?
A: Tasks were distributed based on individual strengths and expertise:
  • One team member focused on API integration and backend logic.
  • Another handled frontend design and user interface development.
  • A third member was responsible for testing and documentation.
Q: How did you manage your time during the project?
A: Time management was crucial. We:
  • Set clear milestones and deadlines for each phase.
  • Used tools like Trello to track progress.
  • Prioritized tasks based on their importance and complexity.
Q: What strategy did you use for documentation?
A: We followed a structured approach:
  • Created a detailed README file explaining the project setup and usage.
  • Documented all API endpoints and their functionalities.
  • Included a project summary and FAQs on the "About" page.
