Frequently Asked Questions

Q: How did you break down the project?
A: The project was broken down into five main phases:
  • Phase 1 - API Selection and Testing (1 week):
    • Researched and selected Spotify and Lyrics.ovh APIs
    • Tested endpoints with Postman
    • Documented API limitations and responses
    • Validated the choice with the supervisor
  • Phase 2 - Design and Wireframing (1 week):
    • Created wireframes for all required pages
    • Designed the user interface with a music-focused theme
    • Validated the site structure
  • Phase 3 - Development (2 weeks):
    • Implemented song search functionality using Spotify API
    • Developed filtering system for date, artist, and duration
    • Created pagination for search results
    • Built favorites system with JSON persistence
    • Integrated lyrics retrieval from Lyrics.ovh
  • Phase 4 - Testing and Debugging (3 days):
    • Tested all functionalities
    • Implemented error handling for API failures
    • Tested browser compatibility
    • Fixed identified bugs
  • Phase 5 - Documentation and Finalization (4 days):
    • Wrote comprehensive README
    • Documented all routes
    • Created this FAQ page
    • Prepared the final deliverable
Q: How did you manage your time?
A: Time management was structured around several key aspects:
  • Planning:
    • Set weekly progress checkpoints
    • Built in a 3-day buffer for unexpected issues
    • Prioritized tasks at the beginning of each week
    • Alternated between focused work sessions and breaks
  • Prioritization:
    • First focused on the mandatory features (search, filtering, pagination, favorites)
    • Developed features based on their technical dependencies
    • Implemented continuous testing to prevent bug accumulation
    • Only added bonus features after completing the essentials
  • Daily Organization:
    • Dedicated 2-4 hours per day to development
    • Reserved afternoons for testing and debugging
    • Documented progress at the end of each day
    • Performed daily code reviews
  • Risk Management:
    • Identified critical points early (API token management, error handling)
    • Prepared backup solutions for complex features
    • Monitored API limitations and rate limits
    • Maintained regular progress tracking

Q: What challenges did you face and how did you overcome them?
A: Several challenges were encountered during development:
  • API Token Management:
    • Challenge: Handling expired Spotify access tokens
    • Solution: Implemented automatic token refresh with expiry tracking
  • Lyrics Retrieval:
    • Challenge: Inconsistent responses from Lyrics.ovh API
    • Solution: Added error handling and sanitization for search queries
  • Pagination System:
    • Challenge: Maintaining state between search, filter, and page navigation
    • Solution: Used URL parameters to preserve state across page loads
  • Error Handling:
    • Challenge: Gracefully handling API outages
    • Solution: Implemented custom error pages and fallback content
  • Performance Optimization:
    • Challenge: Slow response times for search results
    • Solution: Implemented client-side filtering and caching of recent searches
